```sql
SELECT v.table_schema,

   OR v.view_definition ILIKE '%audit_instance%'
   OR v.view_definition ILIKE '%audit_step_def%'
   OR v.view_definition ILIKE '%audit_step_instance%';
```

人工确认：

- 返回结果里不再引用本次计划删除字段

### 4.2 函数 / 存储过程依赖

```sql
SELECT n.nspname AS schema_name,
       p.proname AS function_name,

   OR pg_get_functiondef(p.oid) ILIKE '%audit_instance%'
```

## 5. 数据回放检查

删列前需要抽样确认历史数据不再需要这些字段参与页面展示和流程判断。

### 5.1 抽样检查 `audit_step_def`

```sql
SELECT id, flow_id, step_name, step_no, step_type, role_id
FROM audit_step_def
ORDER BY flow_id, step_no NULLS LAST, id
LIMIT 100;
```

确认点：

- `step_type` 能表达节点类型
- `role_id` 能表达审批规则

### 5.2 抽样检查 `audit_step_instance`

```sql
SELECT id, instance_id, step_def_id, step_index, step_name, status, approver_id, approver_org_id, comment, approved_at, created_at
FROM audit_step_instance
ORDER BY id DESC
LIMIT 100;
```

确认点：

- 页面和接口只需要这套字段即可完成展示与待办判断

### 5.3 抽样检查 `audit_instance`

```sql
SELECT id, flow_def_id, entity_type, entity_id, status, requester_id, requester_org_id, started_at, completed_at, created_at, updated_at
FROM audit_instance
ORDER BY id DESC
LIMIT 100;
```

确认点：

- 当前节点定位已不再依赖 `current_step_index`
- 页面不再依赖 `title`、`result`

### 5.4 历史数据与前端展示确认

如果生产库中仍存在历史实例数据，需要额外确认被删字段不会影响页面展示和历史回放。

重点字段：

- `audit_instance.title`
- `audit_instance.result`

确认方式：

```sql
SELECT id, entity_type, entity_id, status, title, result, started_at, completed_at
FROM audit_instance
WHERE title IS NOT NULL
   OR result IS NOT NULL
ORDER BY id DESC
LIMIT 100;
```

人工确认：

- 前端实例列表、实例详情、审批历史不再直接读取 `title`、`result`
- 当前页面展示已能用 `entity_type + entity_id`、节点实例信息、状态信息替代这些字段
- 若仍有前端页面直接展示 `title` 或 `result`，必须先完成前端切换，再执行删列
