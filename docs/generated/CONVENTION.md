# `docs/generated` 使用约定

本文件用于正式约定：**后续由 AI 或脚本自动生成的后端设计、排查、汇报、清单类文档，默认统一存放到 `docs/generated/`。**

## 适用范围

以下内容默认放入 `docs/generated/`：

- **修复设计**：某个模块的缺陷修复方案、整改设计、风险收口方案
- **性能排查**：接口性能报告、慢链路分析、查询放大分析、优化建议
- **自动汇报**：阶段性检查结果、联调问题总结、巡检报告、自动审计结果
- **执行清单**：回归建议、上线前检查清单、优化优先级清单
- **可重建产物**：分片正文、索引清单、脚本生成的清单文件

## 不适用范围

以下内容**不应默认放入** `docs/generated/`：

- **正式基线文档**：长期有效、需要人工持续维护的权威设计文档
- **核心规范**：会被研发团队反复引用的架构规范、数据库规范、接口规范
- **业务运行数据**：审批记录、消息记录、日志明细、业务表快照
- **纯临时草稿**：尚未成型且不需要沉淀的零散笔记

这类内容应分别放在：

- `docs/` 根目录：正式基线文档
- `archive/`：已完成但需要留档的历史材料
- 数据库 / 日志系统 / 对象存储：运行期业务数据

## 默认生成规则

后续若无特殊说明，自动生成文档遵循以下规则：

1. **先落 `docs/generated/`**，不要直接写入 `docs/` 根目录。
2. **按主题建子目录**，例如：
   - `performance-reports/`
   - `fix-designs/`
   - `audit-reports/`
   - `release-checklists/`
3. **优先按模块 + 日期组织**，避免不同轮次内容互相覆盖。
4. **长文档优先分片保存**，并生成 `manifest.json`。
5. **需要完整阅读时再重建全文**，避免长期保存重复副本。

## 命名建议

建议统一使用英文目录 + 明确主题：

- `performance-reports/message-center/2026-04-26/`
- `fix-designs/workflow-engine/2026-04-26/`
- `audit-reports/permissions/2026-04-26/`

分片文件建议：

- `chunks/0001-summary.md`
- `chunks/0002-current-state.md`
- `chunks/0003-risks.md`
- `chunks/0004-solution.md`

## 文档晋升规则

当 `docs/generated/` 中的某份文档满足以下条件时，可以考虑“晋升”为正式文档，迁移到 `docs/` 根目录：

- 内容已经稳定，不再频繁自动改写
- 已被确认是团队共识方案
- 后续会被长期引用和人工维护
- 不再只是某一轮分析结果，而是正式规范或基线设计

## 当前结论

**从现在开始，`docs/generated/` 作为后端自动生成文档的默认管理区正式启用。**

除非你明确指定要写入 `docs/` 根目录，否则我后续生成的后端修复设计、性能报告、排查汇报、检查清单，都会默认按这套规范落到 `docs/generated/`。
