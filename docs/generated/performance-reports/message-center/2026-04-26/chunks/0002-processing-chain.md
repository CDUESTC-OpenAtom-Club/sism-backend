## 02 当前处理链路

### 模块职责

消息中心当前承接四类能力：

- `summary`：返回待审批、未读提醒等摘要计数
- `messages`：返回消息列表
- `detail`：返回单条消息详情
- `read`：处理已读状态更新

### 当前主链路

控制层进入 `MessageCenterApplicationService` 后，核心流程基本是：

1. 拉取当前用户的**工作流待办**
2. 拉取当前用户的**通知消息**
3. 在应用层构造统一消息对象
4. 在内存中做去重、分类过滤、关键字过滤、风险过滤
5. 按时间排序后，再手工分页返回

### 这条链路的问题

- **分页太晚**：数据已经全量拉到内存里，数据库和查询层的分页优势没有发挥出来。
- **聚合太重**：同一条业务数据在多个接口里被重复组装。
- **依赖太深**：消息中心刷新一次，会连带调用工作流读模型、通知仓储以及补充信息查询。

这意味着当前设计更像一个“全量聚合器”，而不是“按条件读取当前页”的读模型接口。
