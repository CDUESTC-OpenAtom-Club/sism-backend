## 06 修复优先级与验收建议

### 风险优先级

- **P0**：异常信息直出，立即整改
- **P1**：认证、指标、角色等入口的 DTO 与控制器校验基线统一
- **P1**：清理动态 SQL 标识符拼接
- **P2**：生产环境 Swagger / Actuator 暴露面治理
- **P2**：数据库最小权限与默认凭据治理

### 验收建议

整改完成后，建议至少执行以下检查：

- 构造空值、超长值、非法枚举、恶意特殊字符，对核心 API 做输入回归
- 对异常场景回包做断言，确认响应体不再出现内部异常明文
- 全仓库检索 `getMessage()` 返回到 `ApiResponse.error(...)` 的路径，确认已收敛
- 全仓库检索 `@RequestBody` 是否存在未配套 `@Valid` 的关键入口
- 对数据库账号做一次实际授权核验，确认不具备 DDL 或超范围读写权限

### 本次最终判断

如果只问“当前是否存在明确可打的高危注入链”，本次结论是：**暂未发现。**

如果问“当前是否已经达到比较稳妥的上线安全基线”，结论则是：**还没有。** 在异常信息输出、统一输入校验和配置硬化完成之前，这个后端仍应视为存在明确的安全整改事项。
