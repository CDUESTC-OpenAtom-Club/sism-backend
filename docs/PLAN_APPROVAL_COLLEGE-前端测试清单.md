- 因此最后一步 `职能部门终审人审批`，推荐使用教务处审批账号测试。

默认密码: `admin123`

## 一、流程说明

步骤如下:
1. 填报人提交
2. 二级学院审批人审批
3. 学院院长审批人审批
4. 职能部门终审人审批

推荐测试账号:
- 发起人: `jisuanji_report`
- 第二步审批人: `jisuanji_audit1`
- 第三步审批人: `jisuanji_leader`
- 第四步审批人: `jiaowu_audit1`

## 二、发起操作

1. 使用 `jisuanji_report / admin123` 登录。
2. 进入 `计划管理`。
3. 找到计算机学院对应的草稿 Plan。
4. 点击 `提交审批` 或 `发起审批`。
5. 弹窗中点 `确认`。
6. 记录 Plan 名称。

预期结果:
- 第一步自动完成。
- 第二步待办分配给 `jisuanji_audit1`。

## 三、第二步审批

1. 退出后使用 `jisuanji_audit1 / admin123` 登录。
2. 进入 `审批中心` 或 `我的待办`。
3. 找到该 Plan。
4. 点击 `通过`。
5. 输入 `流程4第二步通过`。
6. 点击确认。

预期结果:
- 第二步通过。
- 第三步待办分配给 `jisuanji_leader`。

## 四、第三步审批

1. 退出后使用 `jisuanji_leader / admin123` 登录。
2. 进入 `审批中心` 或 `我的待办`。
3. 找到同一条 Plan。
4. 点击 `通过`。
5. 输入 `流程4第三步通过`。
6. 点击确认。

预期结果:
- 第三步通过。
- 第四步待办分配给 `jiaowu_audit1`。

## 五、第四步审批

1. 退出后使用 `jiaowu_audit1 / admin123` 登录。
2. 进入 `审批中心` 或 `我的待办`。
3. 找到同一条 Plan。
4. 点击 `通过`。
5. 输入 `流程4终审通过`。
6. 点击确认。

预期结果:
- 流程结束。
- Plan 状态变为已审批。

## 六、驳回测试

建议至少测两种:

场景 A: 第二步驳回
1. `jisuanji_audit1` 点 `驳回`。
2. 输入 `流程4第二步驳回`。

预期:
- 流程退回发起人。

场景 B: 第四步驳回
1. `jiaowu_audit1` 点 `驳回`。
2. 输入 `流程4终审驳回`。

预期:
- 流程退回学院侧。
- `jisuanji_leader` 或上一步相关账号重新可见待办。

## 七、前端检查点

- 第二步待办应只落到学院审批人。
- 第三步待办应只落到学院负责人。
- 第四步待办应落到职能部门审批人，不应落到战略发展部账号。
- 全过程不需要手工选择审批人。
