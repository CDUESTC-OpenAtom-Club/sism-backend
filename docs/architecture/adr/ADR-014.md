ADR-014: Migration plan for dual SysUser/User repositories

Context:
- ADR-002 introduced a dual repository pattern (SysUserRepository and UserRepository).
- Current approach complicates maintainability and introduces conflicting ownership.

Decision:
- Propose a phased deprecation and consolidation plan with minimal risk.
- Phase 1: Audit usage, annotate ownership, and prepare for migration.
- Phase 2: Introduce a unified repository interface for SysUser/User domain, while keeping both  repositories for compatibility during migration.
- Phase 3: Remove redundant repository in a follow-up iteration after comprehensive tests.

Consequences:
- Establishes clear ownership and a path toward a single domain-oriented repository model.
- Requires careful regression testing and database interactions to avoid breaking changes.
