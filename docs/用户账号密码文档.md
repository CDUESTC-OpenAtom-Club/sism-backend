## 说明

1. 当前种子数据为每个组织都补齐了可登录测试账号。
2. 角色体系以审批工作流为准，只保留 4 个核心角色：
   - `ROLE_REPORTER` 填报人
   - `ROLE_APPROVER` 审批人
   - `ROLE_LEADER` 领导
   - `ROLE_FINAL_APPROVER` 终审人
3. 职能部门默认提供 4 个账号：`report` / `audit1` / `leader` / `final`
4. 二级学院默认提供 3 个账号：`report` / `audit1` / `leader`
5. 战略发展部保留独立管理员、领导和终审账号。
