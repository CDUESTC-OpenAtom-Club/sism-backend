# PLAN_APPROVAL_FUNCDEPT 前端测试清单

适用流程: `Plan审批流程（职能部门）`

流程定义来源:
- [audit_step_def-data.sql](/Users/blackevil/Documents/前端架构测试/sism-backend/database/seeds/audit_step_def-data.sql#L39)

默认密码: `admin123`

## 一、流程说明

步骤如下:
1. 填报人提交
2. 职能部门审批人审批
3. 分管校领导审批
4. 战略发展部终审人审批

推荐测试账号:
- 发起人: `keji_report`
- 第二步审批人: `keji_audit1`
- 第三步审批人: `keji_leader`
- 第四步审批人: `zlb_final1`

说明:
- 第三步“分管校领导审批”按发起部门自己的 `ROLE_VICE_PRESIDENT` seat 解析，所以科技处流程这里应使用 `keji_leader`，不是统一 `admin`。

## 二、发起操作

1. 使用 `keji_report / admin123` 登录。
2. 进入 `计划管理`。
3. 找到科技处对应的草稿 Plan。
4. 点击 `提交审批` 或 `发起审批`。
5. 在确认框里点击 `确认`。
6. 记住 Plan 名称。

预期结果:
- 第一步自动完成。
- 第二步待办分配给 `keji_audit1`。

## 三、第二步审批

1. 退出后使用 `keji_audit1 / admin123` 登录。
2. 进入 `审批中心` 或 `我的待办`。
3. 找到该 Plan。
4. 点击 `通过`。
5. 输入 `流程3第二步通过`。
6. 点击确认。

预期结果:
- 第二步完成。
- 第三步待办出现。

## 四、第三步审批

1. 退出后使用 `keji_leader / admin123` 登录。
2. 进入 `审批中心` 或 `我的待办`。
3. 找到同一条 Plan。
4. 点击 `通过`。
5. 输入 `流程3第三步通过`。
6. 点击确认。

预期结果:
- 第三步完成。
- 第四步待办分配给 `zlb_final1`。

## 五、第四步审批

1. 退出后使用 `zlb_final1 / admin123` 登录。
2. 进入 `审批中心` 或 `我的待办`。
3. 找到同一条 Plan。
4. 点击 `通过`。
5. 输入 `流程3终审通过`。
6. 点击确认。

预期结果:
- 全流程结束。
- Plan 状态变为已审批。

## 六、驳回测试

建议分别测两种:

场景 A: 第二步驳回
1. `keji_audit1` 点 `驳回`。
2. 输入 `流程3第二步驳回`。

预期:
- 流程回退给发起人。

场景 B: 第四步驳回
1. `zlb_final1` 在终审点 `驳回`。
2. 输入 `流程3终审驳回`。

预期:
- 流程回到前一审批阶段。
- `keji_leader` 或上一步相关处理人重新可见待办。

## 七、前端检查点

- 只有前一步通过，下一步待办才会出现。
- 四个步骤名称应和流程表一致。
- 审批历史里应能看到完整四步链路。
