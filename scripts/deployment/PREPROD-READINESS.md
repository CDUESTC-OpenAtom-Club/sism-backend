# SISM 预发布标准说明

## 目标

本文件用于定义“从候选生产版本提升到预发布标准”的最低要求。

## 预发布标准

必须满足：

1. 前端测试通过且构建通过
2. 后端 CI 为阻塞式测试
3. 发布流程包含 Flyway `validate` 与 `migrate`
4. 健康检查以 `HTTP 200` + `status=UP` 为成功标准
5. Swagger / OpenAPI 默认关闭
6. Actuator 仅允许受控访问
7. 数据库回滚 SOP 已存在
8. 发布/回滚检查清单已存在

- 前端构建：已满足
- 前端测试：已满足
- 后端 CI 阻塞：已满足
- Flyway 发布步骤：已满足
- 健康检查判定：已满足
- Swagger 默认关闭：已满足

## 当前结论

代码与文档层面：`已达到预发布标准`

环境层面仍需人工确认：

1. 线上 nginx 已按 `docs/nginx/sism.conf` 收紧
2. `/opt/sism/.env` 已按生产值配置
3. 实际部署后 smoke 与 UAT 已通过

在未完成以上 3 项前，不应宣称“已正式进入生产运行态”。
